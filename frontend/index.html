Some info about this project


  • This is PicCap-Plus, a fork of PicCap/hyperion-webos with its own additions on top of the original project described below.

  • Direct-to-WLED output: hyperion-webos can now compute the per-LED colors itself and send them straight to a WLED controller via DDP, with no HyperHDR or PC/Pi in between - includes configurable rise/fall smoothing to avoid flicker.

  • An embedded web config UI, reachable at http://<this TV's IP>:8090/ (scan the QR code on the Home tab): full settings, live status, and a HyperHDR-style LED layout editor with a live preview.

  • GitHub: github.com/ICEbAIR82/piccapplus

  • _______________________________________________

  • None of this works without the original PicCap and hyperion-webos projects underneath it. hyperion-webos is the real heavy lifting: it talks to webOS's own capture internals - undocumented and only usable at all thanks to prior reverse-engineering - to grab a live copy of whatever's currently on screen. PicCap is this on-TV app that turns that into something you can actually configure by pointing a remote at your television, instead of editing files over SSH.
    One limitation is inherited as-is and can't really be worked around: webOS refuses to hand over DRM-protected frames from things like Netflix or Amazon Prime Video, while anything arriving over a plain HDMI input is unaffected. So a PC, Fire TV Stick, or Chromecast feeding the TV over HDMI keeps your LEDs working even for DRM'd content.
    As with the original, this app needs to run as root and will try to get there automatically on first launch via the Homebrew Channel.
